1. 范围

此设计规范适用于软件股份智慧人社部门内部平台与系统的设计和开发工作，旨在通过建立 UI 规范，缩减项目设计和开发的时间成本，降低用户在不同系统间的适应难度，确定人社 B 端产品的整体风格。本规范通用于一般项目的开发，也可根据不同项目的需求和特点进行灵活变通。

2. 设计原则

1. 效率优先

减少操作步骤，高频功能一键触达；支持键盘快捷操作（如批量处理）。

2. 一致性

跨模块组件样式、交互逻辑统一，确保用户在系统各处认知一致。

3. 可扩展性

预留业务扩展接口；适配多角色权限视图及复杂业务逻辑的动态增删。

4. 容错性

关键操作二次确认；实时保存草稿，提供操作历史追溯及撤回机制。

5. 操作流畅性

保持用户操作连贯性，不允许产生过多交互层级和冗余特效，确保系统响应灵敏。

4. 间距 (Spacing)

遵循 8px 倍数原则。统一的间距步进能让页面产生有节奏的呼吸感。

8px (XS) - 极小间距，用于图标与文字
16px (S) - 标准间距，用于表单项、列表行
24px (M) - 内部区块间距，用于卡片内间距
32px (L) - 大间距，用于主内容区分割
